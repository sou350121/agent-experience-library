# 跨境 AI 创业：身份/结构/合规/退出路径的决策清单（以 Manus 讨论为引子）

> 目标：把“迁址/切割/并购退出”从八卦叙事提纯为可执行的决策框架。
> 
> 说明：本文不评判对错，只讨论在特定监管环境下的**结构性约束**。

---

## 0. 一句话结论

对跨境 AI 创业而言，**技术只是门票**；能否进入某个市场/资本/并购体系，取决于你是否满足它的“接口契约”（法律、资本结构、数据合规、团队可控性）。

---

## 1. 两条路线（先选体系，别想左右逢源）

### 路线 A：本土体系
- 面向本土市场、资本与规则
- 优势：落地阻力小
- 代价：海外融资/并购/上市空间可能受限

### 路线 B：全球体系
- 面向全球市场、资本与规则
- 优势：更容易获得全球用户/收入/并购机会
- 代价：早期就要承担法律、合规、数据、团队国际化的成本

---

## 2. 结构化 checklist（给创始人/PM/法务/投资人）

### 2.1 产品与市场（决定你被谁需要）
- [ ] 你的付费主力市场在哪里？（北美/欧洲/东南亚/国内）
- [ ] 你的产品是否依赖某些地区不可用的模型/工具？
- [ ] 你的增长路径是否必须靠海外社区口碑？

### 2.2 法律与实体（决定你归哪个司法体系）
- [ ] 总部与主体公司设在哪个司法辖区？为什么？
- [ ] 董事会/治理结构是否清晰、可审计？
- [ ] 关键合同（客户/供应商/模型服务）是否可在目标法域执行？

### 2.3 资本结构（决定谁能买你、谁能投你）
- [ ] cap table 是否“干净可解释”？是否存在未来交易对手不愿接受的关联？
- [ ] 是否需要设立国际业务主体与本土业务主体，并明确边界？
- [ ] 关键知识产权（IP）归属在哪个主体？是否可迁移/可授权？

### 2.4 数据与合规（决定你是否能进入并购方体系）
- [ ] 数据分类：哪些是个人数据/敏感数据/企业机密？
- [ ] 数据跨境：是否存在跨境传输与存储的合规要求？
- [ ] 权限与审计：关键系统是否可做审计日志与访问控制？

### 2.5 团队与运营（决定整合成本）
- [ ] 核心研发团队在哪里？关键岗位是否可迁移/可替代？
- [ ] 关键权限（代码仓库/云账号/证书）是否可收口到目标体系？
- [ ] 若发生并购整合，团队/系统/数据的“交接清单”是否可一次性完成？

---

## 3. 并购视角（买方真正关心什么）

- [ ] **不确定性**：是否存在难以评估的监管/合规风险
- [ ] **可控性**：数据、权限、人才、资产是否可被纳入买方体系
- [ ] **可审计性**：历史交易、股权、合同与资产是否清晰
- [ ] **整合成本**：并购后要花多久才能把它变成“内部可维护”的系统

---

## 4. 把它落到 PRD / Plan Mode

当你让 Agent 帮你做产品规划时，建议在 PRD 增加一个小节：

- **体系选择**：路线 A 还是路线 B？
- **合规边界**：哪些能力/数据不能碰？
- **退出假设**：你希望未来 2-3 年的可能退出方式是什么（并购/上市/现金流）？

这会显著减少后期“结构性返工”。
