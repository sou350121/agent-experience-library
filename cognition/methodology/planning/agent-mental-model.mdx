# Agent 的心智模型：从“做题家”到“管理者”

理解 Agent 的运作机制，是构建复杂自动化系统的起点。

## 1. 核心公式：Agent = LLM + 外骨骼

如果将 LLM 比作一个博学但“残疾”的大脑，那么 Agent 就是赋予它行动能力的“外骨骼”：
- **大脑 (LLM)**：负责理解意图、规划路径、决策。
- **手脚 (Tools)**：负责执行具体任务（联网、写代码、抓数据、查数据库）。

## 2. 思维转换：管理思维 vs 编程思维

构建 Agent 时，开发者应当扮演“管理者”而非“程序员”的角色：
- **不关注过程，关注职责 (Scope)**：不需要告诉 Agent 每一行代码怎么写，而是定义它的工作职责和权限。
- **授权模型**：将工具集（Tools）“授权”给 Agent，并提供一套详尽的“操作手册”（System Prompt）。

## 3. 设计原则：原子化与成本意识

### 拒绝“大而全”
超级全能的 Agent 会导致：
1. 逻辑混乱。
2. 运行效率低下。
3. Token 成本飙升。

### 阶梯化人才选型
像招聘一样选择模型：
- **简单/验证任务**：使用 `GPT-5-mini`（应届生成本）。
- **复杂重构/规划**：使用 `GPT-5.2 Codex`（资深专家成本）。

## 4. 最小可行性案例 (MVP)：X 数据研究员

一个高效的 Agent 研究员应具备以下结构：
1. **身份定义**：拥有 X 搜索权限的资深调研员。
2. **工具箱**：`search_x` API, `get_user_profile` API。
3. **任务下达**：清晰的指令（如：找出最近 30 天聊 Vibe Coding 相关话题最多的 10 个账号）。
4. **自主执行**：搜索 -> 过滤 -> 统计 -> 总结。

---
> 💡 **总结**：AI 时代真正的生产力飞跃，不在于写出完美的 Prompt，而在于构建能自主干活的 Agent 军团。
