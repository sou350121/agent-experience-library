# 10,924 倍的炸彈：DeepSeek mHC 複現報告與架構穩定性稅

> **信號**：FlowMode 工程師 Taylor Kolasinski 成功複現 DeepSeek mHC，在 1.7B 規模下觀察到驚人的信號爆炸，並證實了 Sinkhorn 約束的必要性。

---

## 🚀 提純後的核心觀察

### 1. 超連接 (HC) 的內在不穩定性
- **現象**：在 1.7B 參數規模下，不受約束的混合矩陣（HC）會將信號放大 **10,924 倍**（DeepSeek 論文報告 27B 下為 3000 倍）。
- **根因**：HC 破壞了殘差連接的「恆等映射」屬性，混合矩陣在激進學習率下會不斷累積增益，形成數值爆炸。

### 2. Layer 0 是「金絲雀」
- **發現**：不穩定性率先發生在 **第 0 層**。因為第 0 層直接面對原始 Embedding，沒有 LayerNorm 把關。如果 Embedding 尺度不匹配，HC 會嘗試通過「放大」來補償，進而引發連鎖反應。

### 3. mHC：強制執行的守恆定律
- **解法**：DeepSeek 引入 Sinkhorn-Knopp 算法，將矩陣約束在 **雙擬隨機矩陣 (Doubly Stochastic)** 流形上。
- **效果**：Amax 始終保持在 1.0。這不是在「學習」穩定，而是通過物理導軌「強制」穩定。

---

## 💡 給實踐者的啟示

1.  **約束優於學習**：在大規模系統中，有些屬性（如能量守恆、信號幅度）不應該交給模型去學習，而應該作為架構約束被硬編碼。
2.  **監控 Amax**：如果你在嘗試新架構，請務必監控每一層的信號放大倍率。一旦超過 10 倍，你就是在積累「定時炸彈」。
3.  **無聊的選擇能規模化**：ResNet 的長壽是因為它「無聊且穩定」。mHC 的偉大之處在於它找到了「表達能力」與「穩定性」的平衡點。
