# Post-Scaling Research Age：把“更聪明地用算力”落到工程清单

> 目标：将“Scaling Age 结束，Research Age 来临”的观点提纯为 **可执行的工程路线图**。

---

## 1. 现象：Scaling 仍重要，但边际收益不再线性

- **问题不是“停止 scaling”**，而是：
  - 训练 compute 与性能之间的关系更不稳定
  - 长尾能力的成本更难承受
  - 仅靠堆料会错过更关键的进展杠杆

参考：
- Sara Hooker, *On the Slow Death of Scaling*：`https://ssrn.com/abstract=5877662`

---

## 2. Research Age 的三条主路（工程视角）

### 2.1 推理时扩展（Inference-time Scaling）
把算力投入从“训练侧梯度更新”转向“推理侧搜索与验证”。

- **Best-of-N / Rerank**：用多个候选 + 选择器提高质量
- **Tool Use**：把计算交给工具（检索、执行、验证）
- **Multi-agent**：拆任务、并行探索、交叉校验

#### 落地 Checklist
- [ ] 明确哪些任务允许 N-sampling（成本上限是多少）
- [ ] 设计 rerank / verifier（规则/模型/混合）
- [ ] 输出必须携带证据（引用或可复现实验）

---

### 2.2 数据空间可塑（Synthetic Data / Targeted Data)
数据从“被动采集”升级为“主动设计”。

- **长尾优先**：为稀缺但高价值场景定制样本
- **评测驱动的数据生成**：用失败用例反推数据缺口

#### 落地 Checklist
- [ ] 建立“失败样例库”（真实 bug / 真实用户对话）
- [ ] 用合成数据补齐长尾，但保留“真实分布”回归集
- [ ] 对合成数据做去重、污染检测、版本化

---

### 2.3 轻量级计算（Compute Light）
在同等算力下榨取更高收益。

- **数据质量 > 数量**：去重、修剪、优先级排序
- **蒸馏**：强模型教弱模型（训练成本下降，部署更轻）

#### 落地 Checklist
- [ ] 数据集去重（Dedup）与采样策略写成可复跑脚本
- [ ] 蒸馏目标定义清楚：能力/风格/安全
- [ ] 蒸馏后用真实任务回归评估（别只看榜单）

---

## 3. 对 Agent 系统的架构含义：系统层成为能力放大器

当模型本身边际收益下降，系统工程会变成主战场：

### 3.1 引入“路由层”
- 意图路由（Intent Routing） -> 语义路由（Semantic Routing）
- 把模型选择 + 安全过滤（脱狱/PII）+ 语义缓存 + 幻觉门控收口到工程层

参考：
- `docs/capabilities/vllm-semantic-routing-deep-dive.mdx`

### 3.2 幂等安全墙（可靠性交付）
- **墙内**：可审计脚本/工具（确定逻辑）
- **墙外**：模型推理（非幂等）

参考：
- `docs/capabilities/ai-coding-agent-architecture.mdx`

## 4. 一页行动表（给工程团队）

- [ ] 先选一个真实场景做 inference-time scaling（如：代码 review / PRD / Debug）
- [ ] 给这个场景加：N-sampling + verifier + 证据要求
- [ ] 把失败样例沉淀为数据集（真实回归集 + 合成补齐）
- [ ] 把路由、安全、缓存做成可审计策略层
- [ ] 把高风险动作收口到幂等脚本（可回放、可回滚）
