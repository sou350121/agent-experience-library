# Eigent：開源多智能體工作流桌面應用 (The Open-Source Cowork)

> 核心使命：解析 Eigent 的多智能體協同機制，探討如何利用開源工具構建本地優先的「賈維斯」系統。

---

## 🧠 核心洞见

> **“Agent 的未來不在瀏覽器，而在文件系統與終端。”**

Eigent 成功將 **CAMEL-AI** 的多智能體理論與 **MCP** 協議相結合，落地為一個具備「執行權」的桌面應用。它代表了從「AI 聊天」向「AI 員工房」的範式轉移。

---

## 1. 架構特色：多智能體編隊 (Multi-Agent Squad)

Eigent 預定義了四大核心 Agent，支持並行任務處理：

| Agent 類型 | 職責邊界 | 核心能力 |
| :--- | :--- | :--- |
| **開發 Agent** | 代碼、終端、腳本 | 執行 Shell 命令, 修復 Bug, 寫代碼 |
| **搜索 Agent** | 信息採集、網頁提取 | 聯網搜索, 數據清洗 |
| **文檔 Agent** | 知識整理、文件管理 | 撰寫 Markdown, 整理 PRD |
| **多模態 Agent** | 視覺、音頻處理 | 看截圖分析 UI, 聽會議錄音 |

---

## 2. 三大技術支柱

### 2.1 本地優先 (Local-First)
- **模型支持**：通過 Ollama 接入 DeepSeek-V3, Qwen-2.5 等開源模型。
- **數據隱私**：所有截圖、日誌、操作記錄保留在用戶本地磁盤。

### 2.2 MCP 協議集成
- Eigent 充當 MCP Host，可以掛載各類 MCP Server。
- 支持 Notion, GitHub, Slack, Google Calendar 等「外部感官」。

### 2.3 並行執行引擎
- 任務不再是 `A -> B -> C` 的串行等待。
- 當你下達一個複雜指令時，Eigent 會同時啟動多個 Agent 協作，大幅降低任務完成的總時延（Latency）。

---

## 3. 快速部署與架構 (Deployment & Architecture)

Eigent 採用前後端分離架構，支持雲端、社區自托管與企業版。

### 3.1 快速啟動 (社區版)
```bash
# 1. 複製倉庫
git clone https://github.com/eigent-ai/eigent.git
cd eigent

# 2. 安裝前端依賴
npm install

# 3. 啟動開發環境
npm run dev
```

### 3.2 後端與依賴更新
對於本地開發，需要獨立處理 Python 依賴（推薦使用 `uv`）：
```bash
# 進入後端目錄
cd backend
uv sync
```
*註：拉取代碼更新後，建議分別執行 `npm install` 與 `uv sync` 以確保環境一致性。*

---

## 4. 進階功能：人機協同與企業級治理

### 4.1 人工介入 (Human-in-the-loop)
當任務遇到不確定性或卡點時，Eigent 不會盲目執行，而是會主動請求人工輸入。這解決了自主 Agent 在關鍵決策點的「脫軌」風險。

### 4.2 版本選型決策
- **雲版本 (Cloud)**：即開即用，由官方託管模型與算力，適合快速體驗。
- **社區版 (Community)**：100% 開源，數據留在本地，適合隱私敏感的個人開發者。
- **企業版 (Enterprise)**：支持 SSO、訪問控制、定制化開發與 SLA 保證。

## 5. 管理學實踐啟發

Eigent 的設計完美契合了我們在 **[Agent 管理學](../agent-management/README.md)** 中提出的原則：
- **角色解耦**：開發、搜索、文檔職責明確。
- **工具注入**：通過 MCP 實現動態知識與能力注入。
- **審計留痕**：作為桌面應用，其操作日誌是天然的審計來源。
