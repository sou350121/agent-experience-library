# PRD（写给工程师与 LLM Agent 的可执行模板）

> 目标：把“想法”变成“可被工程、设计、销售、LLM Agent 同时执行的文件”。
> 核心：PRD 必须明确 **做什么 / 不做什么 / 怎么验收**。

## 2. PRD 的 7 问（最小闭环字段）

### 2.1 用户是谁（为谁解决问题）
- 目标用户：<persona>
- 典型场景：<触发条件 + 使用场景>

### 2.2 核心痛点（不解决会怎样）
- 痛点描述：<具体问题>
- 后果：<损失/风险/成本>
- 频率：<每天/每周/每月>

### 2.3 产品一句话（不可替代性）
- 定位：<我们是谁>
- 差异化：<为什么用户必须选我们>

### 2.4 核心功能（MVP）
- MVP 范围：
  - 功能 1：<输入/处理/输出>
  - 功能 2：<输入/处理/输出>
- 用户闭环：从 <A> 到 <B>，用户完成一次完整任务

### 2.5 非目标（明确不做）
- 不做 1：<明确写出来>
- 不做 2：<明确写出来>

> 作用：防止 scope creep；防止 Agent 自动“补全需求”。

### 2.6 成功指标（可量化）
- 指标 1：<例如：转化率/留存/耗时/成本>
- 阈值：<数值>
- 时间窗：<例如：上线后 14 天>

### 2.7 风险与边界（失败条件）
- 风险 1：<技术/合规/运营>
- 边界 1：<什么情况算失败/不可接受>
- 缓解：<如何降低风险>

## 4. 给 LLM Agent 的执行约束（强烈建议贴在 PRD 末尾）

```text
你是编码代理。

你必须：
- 先输出 plan（文件清单 + 风险 + 验证方式），我确认后再改代码。
- 严格按照“非目标”避免越界。
- 所有结论必须可验证（引用文件证据或给出验证命令）。
- 交付必须包含：改动文件列表 + 关键 diff 说明 + 验收步骤。

你禁止：
- 在需求未明确时猜测。
- 顺手重构无关模块。
- 修改公共接口而不声明。
```

---

## 5. PRD Prompt（直接复制给 Cursor/Claude Code/Codex）

```text
你是产品经理 + 技术负责人。
请基于我提供的背景信息，输出一份“写给工程师与 LLM Agent 的可执行 PRD”。

要求：
- 必须回答 PRD 7 问：用户、痛点、一句话定位、MVP、非目标、成功指标、风险与边界。
- 必须给出验收用例表格（至少 5 条），且可验证。
- 必须包含“给 LLM Agent 的执行约束”。

输入背景：
- 业务背景：<...>
- 用户画像：<...>
- 现状与痛点：<...>
- 约束：<时间/人力/技术栈>

输出：Markdown。
```
