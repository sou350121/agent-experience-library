# 00 启程：认识 Agent 并装好你的“神笔”

在进入心法之前，我们必须先弄清楚：**为什么有了 ChatGPT，我们还需要 Cursor？** 

很多小白会觉得：“我不就在网页上问问代码，然后复制过来吗？”——这就是你和 10x 工程师之间最大的鸿沟。

## 1. 降维打击：Cursor vs. ChatGPT

如果你把写代码比作**盖房子**，那么两者的区别如下：

| 维度 | ChatGPT (网页版) | Cursor (AI Agent) |
| :--- | :--- | :--- |
| **身份比喻** | **场外顾问** (只能在电话里教你) | **驻场总工** (就在工地，有钥匙，能动手) |
| **项目上下文** | **盲人摸象**：它只知道你粘贴给它的那一小段。 | **全知视角**：它读过你项目里**所有**文件夹和文件。 |
| **操作权限** | **动口不动手**：只给代码块，你自己复制。 | **直接接管**：它能帮你新建文件、删文件、改代码。 |
| **协作方式** | **来回倒腾**：你得在浏览器和编辑器间切 100 次。 | **原生合一**：对话框就在代码旁边，一键应用。 |

### 为什么“上下文 (Context)”是生死线？
当你的项目超过 3 个文件时，ChatGPT 就会开始胡言乱语，因为它不知道文件 A 里的变量在文件 B 里是怎么定义的。而 Cursor 实时索引你的全量代码，它知道你每一个改动的“牵一发而动全身”。

## 2. 什么是 AI Agent (智能体)？

AI Agent 的核心公式是：**大脑 (LLM) + 感官 (读文件) + 手脚 (写文件/跑终端)**。

```text
    [ 网页版 ChatGPT ]                [ Cursor / Claude Agent ]
    
         大脑 (LLM)                        大脑 (LLM)
             |                                 |
         (只给建议)                      ( 计划 + 执行 )
             |                        /        |        \
             v                       v         v         v
         [ 文本回复 ]            [ 建文件 ] [ 写代码 ] [ 跑终端 ]
             |                       |         |         |
         (你自己复制)             ( 自动完成，你点 Save 即可 )
```

## 3. 安装与配置：开启 Pro 级的“神笔”

1.  **下载**：去 [cursor.com](https://www.cursor.com/) 下载并安装。
2.  **继承习惯**：它本质是 VS Code 的强化版，如果你用过 VS Code，安装时选“Import Extensions”，无缝衔接。
3.  **核心配置 (小白必做)**：
    -   按下 `Ctrl + Shift + J` 进入设置。
    -   **Models**：务必开启 **Claude 3.5 Sonnet**（它是目前编程领域的“标准答案”）。
    -   **Cursor Prediction**：开启，它能预判你下一行想写什么。

## 4. 必杀技：Composer (Ctrl + I) —— 真正的 Agent 体验

别再只用普通的聊天窗口了。**Ctrl + I** 才是 Cursor 的灵魂。

-   **普通 Chat (Ctrl+L)**：像是在问老师问题。
-   **Composer (Ctrl+I)**：像是给下属下达指令。你可以说：“把整个项目的按钮都改成蓝色”，它会瞬间修改 10 个文件，并让你一键 **"Save All"**。

## 5. 最小练习：见证奇迹的 1 分钟

1.  打开一个空文件夹，按下 **Ctrl + I**。
2.  输入指令：
    > “帮我写一个简单的网页，中间有一个红色的圆，点击它会随机跳动，背景会变色。**请注意像素级还原我的色彩描述。**”
3.  看它如何自动创建文件、编写代码。
4.  点击 **Accept/Save All**，直接在浏览器打开生成的 `index.html`。

**当你不用“复制粘贴”就看到了运行结果，你就真正理解了什么是 AI 时代。**
