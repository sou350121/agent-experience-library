# 工具工作流：Agent 产出的“DNA 级”标准

> **核心目标**：将 Agent 从单纯的“代码生成器”升级为“工程交付经理”，通过标准化的输出结构确保任务的可追踪性与高质量交付。

## 1. 什么是“DNA 级”产出标准？

由 Victor Wu (@victor_wu_eth) 提炼。这套标准通过 `Agent MD` 指令集深度调教模型，使其产出具备极强的工程闭环能力。

### 核心链条 (The Chain)
1. **结论 (Conclusion)**：先给结果，明确任务是否完成，核心改动点是什么。
2. **分析 (Analysis)**：深入剖析当前问题的底层逻辑、潜在风险及架构考量。
3. **方案 (Solution)**：给出具体的技术方案，包括选用的库、设计的接口等。
4. **里程碑拆分 (Milestones)**：将复杂任务拆解为可量化、可核对的微小步骤。
5. **提交 PR (Pull Request)**：最终执行代码修改并形成高质量的提交记录。

## 2. 为什么需要这个结构？

- **对抗幻觉**：强制模型先分析、再拆解，能有效减少直接写代码时产生的逻辑漏洞。
- **可复查性**：管理者或开发者可以迅速通过“结论”和“里程碑”定位进度。
- **Vibe Coding 友好**：在意图驱动开发中，明确的里程碑是维持 Agent 长期一致性的关键。

## 3. 实战配置建议

在你的 `.cursorrules` 或 Agent 系统提示词中，可以加入以下指令：

```markdown
# Agent 交付规范
每次完成任务或输出方案时，必须遵循以下结构：
1. [结论]：一句话概括结果。
2. [深度分析]：分析代码现状与变更逻辑。
3. [技术方案]：详细说明改动路径。
4. [执行里程碑]：
   - [ ] 步骤 1: ...
   - [ ] 步骤 2: ...
5. [PR 准备]：生成符合规范的提交日志。
```

## 4. 配合模型：GPT-5.2 xhigh

这种高度结构化的工作流特别适合 **GPT-5.2 xhigh** 等具备极强推理能力（Thinking）的模型。它们能完美理解里程碑之间的依赖关系，并在分析阶段预判到潜在的工程坑点。
