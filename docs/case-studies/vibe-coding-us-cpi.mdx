# 案例复盘：Vibe Coding 驱动的宏观数据分析工具

> **来源**：XinGPT (@xingpt) 的实战案例

## 背景与需求
开发者需要分析美国 BLS（劳工部）网站上复杂的 CPI 子项数据，任务涉及 130 多个 Series ID 的调用、同比涨幅计算以及最终的可视化展示。

## 执行过程 (Vibe Coding 模式)

### 阶段一：意图描述
开发者向 Agent（如 Cursor/Claude Code）描述了目标：
- “帮我对接 BLS API，抓取 130 个 CPI 子项。”
- “计算 2025 年最新的同比涨跌幅。”
- “生成一个现代感、卡片式的可视化图表。”

### 阶段二：自动化生成
Agent 自动处理了：
1. **API 批处理逻辑**：自动分批请求以规避 API 限制。
2. **多语言映射**：将 BLS 的 Series ID 自动翻译为中英文标签。
3. **视觉风格定义**：应用了 Nano Banana 审美风格进行可视化。

## 关键发现
| 类别 | 典型项目 | 趋势 | 归因 |
| :--- | :--- | :--- | :--- |
| **涨幅领跑** | 园艺护理、居家照顾 | 🔼 坚挺 | 人力成本驱动的服务通胀 |
| **跌幅居前** | 鸡蛋、智能手机 | 🔽 显著 | 供应过剩与耐用品结构性通缩 |

## 复盘与反思
- **做对了什么？**：利用 Vibe Coding 绕过了繁琐的 API 文档阅读和数据清洗编码，直接进入“洞察”阶段。
- **Agent 表现评价**：在处理具备标准 API 和明确数学逻辑（YoY 计算）的任务时，Agent 的稳定性极高。

## 可迁移的结论
1. **数据抓取新范式**：对于政府公开数据（BLS, Fed 等），直接通过 Agent 生成抓取脚本已成为效率最高的方式。
2. **洞察优先**：开发者的时间应花在分析“鸡蛋跌幅 -13.2% 反映了什么”上，而不是研究“如何解析 JSON 嵌套数组”上。
