# GPT-5.2 vs Claude Opus 4.5: 复杂工程规划能力对比

> **注意**：本篇为示例性质的对比文档，旨在展示评估维度。

## 结论摘要
- **胜者**：**Claude Opus 4.5**
- **核心差异**：Claude 在处理超长上下文的逻辑一致性上表现更优，而 GPT-5.2 在纯粹的单点突破（创意编码）上更有爆发力。
- **推荐建议**：架构设计与大型重构选 Claude；从零启动小型 Demo 选 GPT。

## 评估维度
| 维度 | GPT-5.2 | Claude 4.5 | 备注 |
| :--- | :--- | :--- | :--- |
| **多文件规划** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | Claude 极少出现逻辑断层 |
| **指令遵循** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 两者均属于顶级水平 |
| **幻觉控制** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | GPT 在引用不存在的库 API 时仍有发生 |
| **代码简洁度** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | GPT 生成的代码更具现代感 |

## 对比证据
### 场景：将一个 10 万行的 Monorepo 從 Webpack 迁移到 Rspack
**GPT-5.2 表现：**
生成速度极快，但在处理复杂的 `alias` 配置时，遗漏了 3 个子包的路径映射。

**Claude 4.5 表现：**
先要求读取了全部 `package.json`，然后给出了一个极其详尽的 12 步迁移清单，并准确预判了 `Sass-loader` 的兼容性问题。
