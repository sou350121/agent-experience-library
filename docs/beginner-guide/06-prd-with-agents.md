# 06 用 Agent 写 PRD：小白一键生成需求说明书（可直接喂给 Agent）

你不需要会写“专业 PRD”，你只需要会回答几个问题。

一句话：**PRD = 把“想法”变成“工程/设计/销售/LLM Agent 都能执行的文件”。**

---

## 0. 这份文档怎么用（最简单）

1. 先看 **第 1 节**，准备材料（只要 5 分钟）。
2. 复制 **第 2 节的《PRD 元提示词》**，粘贴到 Cursor/Claude Code/Codex。
3. 把你的材料填到 `<...>` 里。
4. 让 Agent 输出 PRD；你只需要检查：**非目标** 和 **验收**。

---

## 1. 写 PRD 前，小白只要准备这 8 个“变量”

你可以直接照着填：

- 业务背景：<一句话说明你在做什么>
- 用户是谁：<谁会用？具体一点>
- 使用场景：<什么时候用？在哪用？>
- 核心痛点：<不解决会怎样？损失/麻烦/风险>
- 你的一句话产品定义：<为什么必须是你？>
- MVP 功能：<最小闭环：输入->处理->输出>
- 非目标：<明确不做什么，越具体越好>
- 成功指标：<用数字说话：转化率/留存/耗时/成本/错误率>

> 小白提示：如果你不知道成功指标怎么写，就写“用户完成任务的时间从 X 分钟降到 Y 分钟”。

---

## 2. 《PRD 元提示词》（直接复制给 Agent）

把下面这段整段复制给 Agent，然后把变量填好。

```text
你是产品经理 + 技术负责人 + 项目经理（面向执行）。
请基于我提供的背景信息，输出一份“写给工程师与 LLM Agent 的可执行 PRD（Markdown）”。

硬性要求：
1) PRD 必须回答 7 问：
   - 用户是谁（为谁解决问题）
   - 核心痛点（不解决会怎样）
   - 产品一句话（不可替代性）
   - 核心功能（MVP）
   - 非目标（明确不做）
   - 成功指标（可量化）
   - 风险与边界（失败条件）
2) PRD 必须包含“验收标准”，并给出至少 5 条验收用例表格（输入/操作/期望输出）。
3) PRD 必须包含“给编码 Agent 的执行约束”，要求其先输出 plan，再进入执行。
4) 如果我的信息不足，你必须在 PRD 最前面列出“你需要我补充的最小信息清单”（不超过 8 条）。
5) 语言：简体中文为主，必要时保留英文术语。

输出结构（请严格按此结构）：
- 一句话摘要
- 7 问（逐条回答）
- MVP 详细说明（可用列表或流程）
- 非目标（必须显眼）
- 成功指标（带阈值与时间窗）
- 风险与边界（含缓解策略）
- 验收标准（表格）
- 给编码 Agent 的执行约束（可复制区块）

我的输入背景如下（你要把它们写进 PRD）：
- 业务背景：<...>
- 用户是谁：<...>
- 使用场景：<...>
- 核心痛点：<...>
- 一句话产品定义：<...>
- MVP 功能：<...>
- 非目标：<...>
- 成功指标：<...>
- 额外约束（可选）：<时间/人力/技术栈/合规>
```

---

## 3. 小白验收：你只需要检查 3 件事

- 1) **非目标够不够硬**：有没有把“不做什么”写得足够具体？（防止 scope creep）
- 2) **验收够不够可验证**：是不是能用“输入/操作/输出”验证？
- 3) **风险边界够不够诚实**：有没有写清“什么情况算失败/不可接受”？

---

## 4. 常见坑（小白最容易踩）

- 坑 1：只写“功能列表”，不写“验收”。
- 坑 2：没有“非目标”，导致 Agent 自动加戏。
- 坑 3：成功指标写成形容词（例如“体验更好”），而不是数字。
