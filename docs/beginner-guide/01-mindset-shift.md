# 01 心法：从“写代码”到“指挥 AI”

你不用先变成高手，才能用好 Coding Agent。
相反，你要先换一套“工作方式”，AI 才会稳定给你交付。

## 1. 先接受一个事实：AI 的智能是“锯齿状”的
AI 在某些任务上像天才，在另一些任务上像小学生。
这叫“锯齿状智能（Jagged Intelligence）”。它不是坏，而是形态如此。

```text
 能力值 (Ability)
   ^
   |      /\      /|      /\
   |     /  \    / |     /  \
   |    /    \  /  |    /    \
   |   /      \/   |   /      \
   +------------------------------> 任务领域 (Tasks)
      [天才区] [小白区] [天才区]
```

**对小白最重要的结论是：**
- 你不能用“我问一句，它就应该一次答对”的方式使用 AI。
- 你要用“可验证的小步快跑 + 反复校准”的方式指挥它。

## 2. 你需要掌握的不是“写代码”，而是“三个理解”
很多人把 AI 用不好，不是不会提示词，而是缺少这三个理解：

1.  **对业务的理解（建模）**：你要能说清楚“用户要什么、边界条件是什么”。
2.  **对技术栈的理解（镣铐）**：你要能告诉 AI“只能用什么、不许用什么”。
3.  **对架构的理解（轨道）**：你要能告诉 AI“代码该放哪、模块怎么分”。

> 💡 **小贴士**：这里的“轨道”可以先用大白话理解：参考 [物理导轨（单行道与围栏）](../architecture-governance/00-glossary-for-beginners.mdx)。

## 3. 小白最实用的 3 句话任务卡
每次让 AI 干活，你先写清楚这三句话（非常重要）：

| 字段 | 说明 | 示例 |
| :--- | :--- | :--- |
| **我要解决什么问题** | 一句话描述目标，不写细节 | 把【莲蓬】做成仿生设计海报 |
| **成功长什么样** | 给出可观察的验收标准 | 输出五段横向布局，包含渲染图 |
| **限制条件是什么** | 规定风格、技术栈、黑名单 | 纯白背景；不要霓虹色；中文标注 |

## 4. 小白容易踩的坑
- ❌ **只给目标不给限制**：AI 会放飞自我，然后“越写越远”。
- ❌ **一次让它做太多**：拆成 3-5 个可验证的小步骤。
- ❌ **不做验收**：没有闭环就没有质量。
- ❌ **不让它复述计划**：一定要让 AI 先写 **Plan**，你确认后再开工。

> 📖 **词典同步**：[计划模式（开工前的图纸确认）](../architecture-governance/00-glossary-for-beginners.mdx)

## 5. 最小练习（5 分钟）
请选择一个任务：
- **A 计划**：用本仓库模板生成一张海报（仿生 / Lookbook / 电商 KV 任选其一）。
- **B 计划**：让 AI 帮你写一份“本周学习总结”。

**操作要求**：用“3 句话任务卡”把需求写清楚，发给 AI。

完成后你会发现：**你不需要更会提示词，你需要更会定义问题。**
