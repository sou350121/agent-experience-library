# 架构治理：小白版黑话词典 (Glossary for Beginners)

> **导读：** 觉得“架构治理”、“物理导轨”这些词儿太高大上？别被吓到。其实它们用生活里的例子一讲就通。这份词典就是为了帮你拆掉这些“知识围墙”。

---

## 🏗️ 核心术语：城市规划篇

### 1. 架构治理 (Architectural Governance)
- **大白话：** **城市规划局。**
- **解释：** 如果每个人想在哪盖楼就在哪盖，最后城市肯定乱成一团。架构治理就是定规矩：哪里是商业区（业务逻辑），哪里是绿化带（文档），哪里是下水道（底层设施）。

### 2. 物理导轨 (Physical Rails)
- **大白话：** **单行道与围栏。**
- **解释：** 强制规定哪些文件夹只能放哪些文件。比如：Agent 严禁在“卧室”里修厕所。有了导轨，Agent 就不会把代码乱塞。

### 3. 目录拓扑 (Directory Topology)
- **大白话：** **城市地图。**
- **解释：** 整个项目的文件夹长什么样。一张好的地图能让 Agent 一眼看出：哪儿是家，哪儿是公司，哪儿是医院。

## 🍳 开发逻辑：餐厅后厨篇

### 4. 逻辑契约 (Logical Contracts)
- **大白话：** **餐厅菜单。**
- **解释：** 菜单上写了“宫保鸡丁”，你就得给我鸡肉，不能给我猪肉。逻辑契约规定了：我给你什么数据，你必须还给我什么格式的结果。严禁“货不对板”。

### 5. SSOT (单一事实源)
- **大白话：** **标准北京时间。**
- **解释：** 整个镇子只能有一个钟。如果大家表都不一样，那就全乱了。SSOT 规定：关于某个需求或数据，只能有一个地方说了算，其他地方都是复读机。

### 6. 副作用 (Side-Effects)
- **大白话：** **炒菜时的油烟。**
- **解释：** 你的任务是“炒菜”（计算数据），但如果你炒菜的时候把隔壁的窗户震碎了（偷偷改了别处的数据），这就是副作用。好的架构要尽量让“油烟”被排油烟机带走（隔离副作用）。

## 🛠️ 工程实践：装修维护篇

### 7. 技术债 (Technical Debt)
- **大白话：** **没洗的碗。**
- **解释：** 吃饭（写功能）很快，但如果不洗碗（整理代码），下次就没碗用了。攒多了，后厨就瘫痪了。

### 8. Plan Mode (计划模式)
- **大白话：** **开工前的图纸确认。**
- **解释：** 装修工（Agent）动锤子前，得先拿着图纸跟你说：“老板，我打算在墙上开个窗。”你点点头，他再动。这比拆了墙再补要省钱得多。

### 9. Re-Vibe (重新编排/重构)
- **大白话：** **旧房翻新。**
- **解释：** 补丁叠补丁太丑了。干脆把这面墙拆了，按照最新的审美（架构）重新装一遍。在 AI 时代，这事儿成本极低，别心疼。

---

## 🎨 视觉与前端：审美与还原篇

### 10. 像素级还原 (Pixel-Perfect)
- **大白话：** **照镜子。**
- **解释：** 当你让 Agent 参考一个介面时，如果你说“像素级还原”，就是在告诉它：别给我整那些虚的，间距、颜色、圆角、阴影，必须跟原稿长得一模一样。

---

> **💡 建议：** 当你对 Agent 发指令时，可以直接用这些比喻。
