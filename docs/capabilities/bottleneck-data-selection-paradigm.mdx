# 瓶頸數據選取範式 (Bottleneck Data Selection)

數據不是越多越好，而是越「痛」越好。在機器人與 Agent 開發中，關注「任務瓶頸」是提升效率的唯一捷徑。

## 🧠 核心洞见
> **「數據的注意力機制」。**
> 機器人 90% 的時間都在做無效運動。真正的學習發生在任務即將失敗的瞬間，或即將發生接觸的關鍵點。

## 🎯 數據狙擊策略

1.  **鎖定「接觸時刻」**：
    - 忽略自由空間的移動。
    - 在插銷入孔、撿起物體邊緣的微小窗口（可能只有 20x20 像素）進行高密度採樣。

2.  **VLM 作為「數據管理員」**：
    - 利用大模型的語義理解能力，對海量無標注視頻進行「特徵檢索」。
    - **Prompt 示例**：`"找出所有光照不足且機械臂發生劇烈抖動的片段"`。

3.  **反事實模擬 (Counterfactuals)**：
    - 利用高斯潑濺 (Gaussian Splatting) 構建 4D 內容。
    - 在數字世界中人為改變行人位置、物體重量或光照時間，製造現實中難以遇見的「長尾場景」。

## 🚀 Kiro 自動化 Flow 構思
如何將此流程轉化為 Kiro 的自動化流水線？
- **Hook (onVideoUpload)**: 當新的演示視頻上傳時，觸發 VLM 插件進行語義標註。
- **Hook (onFailureDetect)**: 當測試失敗時，自動截取失敗前 1 秒的數據並標記為「高優先級訓練樣本」。
- **Power (DataCleaner)**: 調用專門的 Power，利用 VLM 自動過濾掉遮挡嚴重或標註錯誤的數據。
