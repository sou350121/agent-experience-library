# AlphaOPT：构建可进化的优化建模经验库

AlphaOPT 提出了一种全新的、自我改进式经验库学习框架，用于将模糊的自然语言需求转化为严谨的数学优化模型。

## 🧠 核心洞见
> **「授人以魚不如授人以漁」。**
> 傳統模型是「背題家」，而 AlphaOPT 構建了一個「外部大腦」。它不只是生成代碼，而是通過「學習-演化」循環，將建模經驗結構化。這標誌著從「微調權重」到「治理知識」的範式轉移。

## ⚙️ 核心機制：雙階段循環
AlphaOPT 通過兩個階段的循環，讓 AI 在建模過程中不斷自我進化：

### 1. 經驗庫學習 (Library Learning)
- **錯誤診斷**：當生成的模型出錯時，與標準程序對比或通過「自探索」尋找正確路徑。
- **知識提純**：將修復方案提煉為四要素：**分類 (Taxonomy)**、**條件 (Condition)**、**解釋 (Explanation)**、**示例 (Example)**。
- **動態合併**：自動清理冗餘，保持經驗庫的精煉。

### 2. 經驗庫演化 (Library Evolution)
- **診斷偏差**：分析經驗在跨任務遷移中的「誤檢」與「漏檢」。
- **邊界優化**：動態修正經驗的觸發條件，使其在複雜場景中更加精準。

## 📂 三大知識維度
- **領域建模 (Domain Modeling)**：業務邏輯與領域特性（如：交通、電力、排產）。
- **通用建模 (General Formulation)**：跨領域的數學模式（如：變量定義、約束構造）。
- **代碼實現 (Code Implementation)**：求解器語法與 API 規範（如：Gurobi, CPLEX, Pyomo）。

## 🚀 實踐價值
- **分布外 (OOD) 泛化**：在未見過的任務中，性能超越傳統微調模型（如在 OptiBench 上提升 7.7%）。
- **可解釋性**：每一行代碼的生成都有據可查，具備極強的審計與干預能力。
- **數據效率**：無需海量標注，僅靠「最優解」即可實現自探索學習。
