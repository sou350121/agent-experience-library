# Agent Playbook：AI 应用生态智能情报库

> **不是新闻聚合，是认知过滤。** 每天从 AI 生态的噪音中，提炼出真正值得工程师关注的信号。

---

## 你为什么需要它

- **每天 50+ AI 工具发布** → 自动收集 + 编辑过滤，只留 5-10 条真正有影响力的
- **官方文档不讲生产踩坑** → 架构深评含真实崩溃场景 + AI 生成代码盲点，工程师视角
- **Twitter 热议一天就消失** → 社交情报每日归档，争议 / 观点 / viral 案例可回溯
- **趋势分析无从验证** → 双周预测附 ✅/❌ 历史追踪，判断力有账可查

---

## 智能分层架构

Agent Playbook 不是一个 RSS 阅读器——它是一个多层认知过滤系统：

| 认知层 | 功能 | 实现 |
|--------|------|------|
| **感知** | 覆盖全域信号源 | RSS 自动采集 · 社区 web 搜索 · GitHub 发布追踪 |
| **评级** | 噪音在进入推理前截断 | 编辑评级 🔴重大 / 🟡值得关注 / 🔵背景参考 |
| **推理** | 不止描述，给出判断 | 三段式 LLM：收集 → agent 推理 → 结构化输出 |
| **专家审核** | 再过一道人工智能关卡 | qwen3.5-plus 专家评分，低于阈值的内容不推送 |
| **社交认知** | 捕捉生态在"想什么" | 单独社交情报管道：观点 / 争议 / viral，非新闻稿 |
| **记忆沉淀** | 可查询的知识资产 | 结构化 Markdown，Git 历史，全文可搜索 |
| **元认知** | 系统反思自身判断力 | 双周预测 + ✅/❌ 验证，正确率有记录 |

---

## 信息为什么更有用

同样是"AI 工具日报"，普通聚合 vs. Agent Playbook：

| 维度 | 普通 RSS 聚合 | Agent Playbook |
|------|-------------|----------------|
| 筛选标准 | 有发布就收录 | 影响力 + 新颖性双重过滤 |
| 内容深度 | 官方描述 / 链接 | 架构评审：生产场景 · 崩溃路径 · 代码盲点 |
| 社区视角 | 无 | 每日捕捉争议 / 观点 / 批评 |
| 信息重复 | 多管道重复 | 跨管道去重（日报 ↔ 精选 互不重叠）|
| 趋势判断 | 无 | 双周预测 + 回顾验证，有正确率记录 |
| 读者类型 | 泛用 | 工程师 + 技术决策者，不写给普通用户 |
