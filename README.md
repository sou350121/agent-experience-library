# Agent Playbook

### AI 应用生态的工程师情报站

每天醒来，AI 生态又变了。新工具发布、架构争论、框架翻车——信息量大到没有人能全看。  
Agent Playbook 帮你做一件事：**把值得看的留下来，把噪音过滤掉。**

不是新闻站，不是 Newsletter，是一个持续运行的认知过滤系统。

## 它解决的问题

**你每天打开 Twitter，看到的是：**  
→ 50 个新工具发布，其中 48 个三个月内消失  
→ 一篇架构文章，讲的是 Demo 环境，不是生产  
→ 热议话题，明天就没人提了，你没有存档

**Agent Playbook 的做法：**

- **每天 50+ 发布 → 过滤到 5 条** — RSS 自动采集 + 双重过滤，只留真正有影响力的
- **官方文档不讲踩坑** — 架构深评专写生产崩溃路径，以及 AI 生成代码在哪里出问题
- **热议转瞬即逝** — 社交情报每天归档，一个月后还能找到当时的争议
- **"AI 趋势"没有验证** — 双周报告做可验证预测，下期必须回来打分 ✅❌，有正确率记录

## 为什么信息质量更高

| 维度 | 普通 RSS / 聚合站 | Agent Playbook |
|------|-----------------|----------------|
| 筛选标准 | 发布了就收录 | 影响力 + 新颖性双重过滤 |
| 内容深度 | 官方描述 + 链接 | 架构评审：崩溃路径 · AI 代码盲点 · 生产建议 |
| 社区视角 | 无 | 专线捕捉：争议 / 观点 / 批评，不是新闻稿 |
| 信息去重 | 多渠道重复 | 日报 ↔ 精选跨管道去重，不重复告诉你同一件事 |
| 趋势判断 | 无追踪 | 双周预测 + 历史验证，判断力有记录可查 |
| 读者定位 | 泛用 | 工程师 + 技术决策者，不写给普通用户 |

## 系统架构

Agent Playbook 背后不是人工编辑，是一套多层认知过滤 pipeline：

| 层 | 作用 |
|----|------|
| **感知层** | arXiv · RSS · GitHub releases · 社区 web 搜索，全自动摄入 |
| **评级层** | LLM 编辑过滤 + 专家评分，低质量内容在推理前截断 |
| **推理层** | 三段式 LLM：`prep → agent → post`，给出判断而非描述 |
| **社交认知** | 独立管道专门追踪观点 / 争议 / viral，不混入工具发布 |
| **记忆层** | 结构化 Markdown 归档到 Git，全文可搜索，永久可查 |
| **元认知** | 双周预测 + ✅/❌ 回顾，系统反思自身判断力 |
